# Code of Conduct

## Our Commitment

The Fermi community is committed to creating a respectful, professional, and collaborative environment for everyone.

We welcome contributions of all kinds, including code, documentation, bug reports, feature proposals, testing, and discussions.

Expected Behavior

Community members are expected to:

- Be respectful and constructive in discussions.
- Provide technical criticism without personal attacks.
- Accept feedback and different opinions professionally.
- Focus discussions on improving the project.
- Help maintain a welcoming and productive environment.

Unacceptable Behavior

The following behavior is not acceptable:

- Harassment or personal attacks.
- Insults, threats, or intentionally hostile behavior.
- Discrimination or hate speech.
- Trolling or deliberately disrupting discussions.
- Sharing private information without permission.
- Repeatedly ignoring project rules or maintainer decisions.

Technical Discussions

Technical disagreements are a normal part of software development.

When discussing design decisions:

- Explain your reasoning with technical arguments.
- Respect decisions made by maintainers.
- Understand that not every proposal will be accepted.

A rejected proposal does not mean the contribution was not valuable.

Enforcement

Project maintainers are responsible for enforcing this Code of Conduct.

Maintainers may remove comments, close discussions, reject contributions, or take other appropriate actions when behavior violates these guidelines.

Scope

This Code of Conduct applies to all project spaces, including:

- GitHub issues.
- Pull requests.
- Discussions.
- Documentation contributions.
- Other official communication channels.

Changes

The maintainers may update this Code of Conduct when necessary to better support the project's community.

Contact

For questions or concerns regarding this Code of Conduct, please contact the project maintainers.
